# Depth and Quality

Use this reference to prevent shallow answers. Apply it to complete answer packs, role-search packs, and any user request that asks for detailed preparation.

## Length Targets

Use these as minimum targets unless the user asks for concise output:

| Question Type | Minimum Depth |
| --- | --- |
| Knowledge question | 500-800 Chinese characters, or 300-500 English words |
| High-frequency core knowledge question | 1000-1500 Chinese characters, or 650-900 English words |
| Scenario question | 900-1400 Chinese characters, or 550-850 English words |
| Project detail question | 700-1200 Chinese characters, or 450-750 English words, excluding fill-in placeholders |
| Strategy expansion question | 1200-2000 Chinese characters, or 750-1200 English words, plus a comparison table |
| Behavioral question | 400-700 Chinese characters, or 250-450 English words |

If the output language is mixed, judge by substance rather than exact count. Do not pad with empty repetition. Expand through mechanisms, tradeoffs, examples, production details, and follow-up preparation.

## Required Sections by Type

Knowledge questions must include:

- Direct answer.
- Background context.
- Detailed mechanism.
- Example or analogy if useful.
- Production or interview relevance.
- Spoken interview answer.
- At least 3 follow-up questions.
- Common mistakes.

Scenario questions must include:

- Provided or assumed scenario.
- Impact assessment.
- Immediate mitigation.
- Diagnosis path.
- Root cause possibilities.
- Data repair or consistency checks when relevant.
- Long-term prevention.
- Communication and postmortem notes for production incidents.
- Spoken interview answer.
- At least 3 follow-up questions.

Project detail questions must include:

- Known facts.
- Missing facts.
- Reusable answer framework.
- Candidate fill-in template.
- Optional example project scenario clearly labeled as fictional.
- Risks of overclaiming.
- Follow-up questions the interviewer may ask.

Strategy expansion questions must include:

- Strategy overview.
- Comparison table.
- When to use each strategy.
- Pros and cons.
- Implementation notes.
- Common failure modes.
- Conservative spoken answer.
- Follow-up questions.

## Anti-Short-Answer Rules

Do not answer a complete pack with only:

- A list of bullet points.
- One paragraph per question.
- Definitions without examples.
- Spoken answers without detailed explanations.
- Follow-ups without answers.

If time or context is limited, split the work into batches and create separate sections or files instead of shrinking the answers.

## Final Review Pass

Before delivering a full pack, verify these items internally. Do not output this checklist in the final document:

- Every major question satisfies the minimum depth target.
- Every major question has a spoken answer.
- Every major question has follow-up questions.
- Scenario assumptions are explicit.
- Project facts are not fabricated.
- Search sources are cited when search was used.
- A Markdown file was created when the environment can write files.
- The final document can be used as a standalone study document.
